# nukemore: 抜け漏れ

着手前に文書の穴を見つける。ターミナル出力のみ、ファイル保存なし、入力言語に合わせる。

## 入力

- パス → Read
- URL (Redmine/GitHub) → WebFetch
- 文章 → そのまま
- 引数なし → 「何を点検しますか?」

## 形式 (固定4項目)

1. **暗黙の前提** — 書かれてないが当然視されていること 1〜3個
2. **足りない情報** — 着手に必要だが欠けているもの 1〜3個
3. **聞くべき質問** — 着手前に確認する具体的な問い 3〜5個 (Yes/No か選択肢付き)
4. **危ない箇所** — 仕様の曖昧さ・矛盾・無理筋 0〜3個

## ルール

- 具体的に — 「曖昧」ではなく「『早く』が日数か時間か不明」
- 重要度順
- 専門用語は日常語に翻訳
- 25行以内
- 見出し `#`、メタ発言、丸ごとコピー、ファイル保存は禁止
